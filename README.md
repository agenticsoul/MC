They then introduce co‑presenters François, Claudia, and Erin, all experienced consultants and Marketing Champions leading various user groups and scheduled to present different days in the series. The main technical content in this episode is delivered by Erin Duncan, who focuses on why Marketing Cloud Growth exists and what it does.[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)

## Why Marketing Cloud Growth exists

A large section explains Salesforce’s acquisition history: tools like Pardot (now Account Engagement) and ExactTarget (now Marketing Cloud Engagement) started as separate platforms with different foundations from Salesforce core. Even though they’ve been integrated into the wider stack, their underlying architectures differ, which can cause scalability and data‑silo issues as customers grow.[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)

Marketing Cloud Growth Edition is presented as a response to this: Salesforce is “re‑founding” the best features of these tools on Salesforce core and Data Cloud, to get a single architecture, better scalability, and deeply shared data. The speakers emphasise that sharing a foundation with core and Data Cloud enables one trusted platform, consistent data across sales/marketing/service, better AI performance, and more consistent personalization across the full customer journey.[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)

## Data Cloud and unified individuals

Because Growth sits on Data Cloud, Erin briefly explains Data Cloud as a customer data platform that unifies data from multiple systems (sales, service, marketing, commerce) into a single “360 view” of the customer and can then activate that data into other tools. A key concept is “unified individuals”: Data Cloud uses identity resolution rules to detect that multiple records in different systems all represent the same person (for example, “Rachel” in Service Cloud, Commerce Cloud, and Marketing), merges them, and shares a harmonised profile back out.[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)

Marketing Cloud Growth consumes these unified individuals so marketers segment and personalise using a single, reconciled profile rather than juggling separate lead/contact records from multiple systems.[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)

The webinar then walks through major Growth Edition capabilities, focusing on how “on core” design and shared tools make life easier for marketers. Key points:[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)

- It is built directly on Salesforce core and tightly integrated with Data Cloud, so its UI and behaviour are familiar to Salesforce users and align with other cloud tools.[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)
    
- It uses Salesforce Flow as the orchestration engine, giving marketers granular control over order of operations, waits, and branching, but through a marketing‑friendly flavour of Flow.[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)
    
- It reuses shared builders and components (email builder, segmentation, etc.) across clouds, so users don’t feel like they are constantly switching to completely different tools.[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)

## Campaigns, flows, and builders

Erin shows how Growth is organised around a “centralised campaign hub.” Marketers start by creating a campaign, then choose from guided campaign patterns like single email, message series, form with landing page, SMS, WhatsApp, blank email, or event‑driven automations. If they prefer, they can still build assets first and link them later.[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)

Key builders demonstrated:

- **Email builder**: A drag‑and‑drop Lightning‑style builder where you apply a brand, add components, insert merge fields, and configure subject line/preheader, with options for Einstein assistance and dynamic content.[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)
    
- **Form builder**: A highly configurable form designer that lets you set placeholders, validations, styling down to padding and margins, choose a data source (e.g. Contact) for automatic field mapping, or leave it blank and map via Flow; it supports splitting submitted data across Contact, Account, Tasks, etc.[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)
    
- **Event‑trigger setup**: For “blank event” flows, the tool offers a growing list of triggers (email events, form submissions, SMS/WhatsApp interactions, and a custom real‑time Data Cloud data graph event) to start automations.[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)
    

Flow is used to orchestrate journeys: send messages, wait a precise time or until an event, branch based on behaviour, and plug in multi‑channel steps like SMS vs WhatsApp, often combined with experimentation features in Advanced Edition.[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)

## Brand guidelines and reusable design

The platform includes a centralized “Brand Guidelines” area where you define brand identity and tone (for Agentforce/Einstien copywriting), as well as colours, typography, buttons, and layout rules used across assets. Teams can have multiple brands, set a default per workspace, and auto‑apply branding to new emails, forms, and pages.[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)

A notable benefit is change propagation: if you later update something like the button colour in the brand guidelines and republish, that change flows down to existing branded assets, avoiding manual edits in many emails.[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)

AI support appears throughout:

The overarching message is that AI leverages the full cross‑cloud dataset in Data Cloud to improve both predictive and generative tasks.[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)

## Personalization, dynamic content, and scoring

Personalization is discussed at several layers:

- **Cross‑object personalization** uses Data Cloud data graphs so an email can pull fields not just from a Contact but from related objects like the most recent closed‑won Opportunity or Account attributes, without pushing all fields onto the Contact.[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)
    
- **Dynamic content** lets marketers define variations (e.g., for existing vs prospective customers, or based on industry) and swap headings, CTAs, and other elements based on rules tied to fields like Account type or status.[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)
    

People scoring (similar to Pardot scoring and grading) provides:

- Engagement score: how much a person interacts with marketing.
    
- Fit score: how well they match the ideal customer profile.[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)
    

Marketers can weight these components (e.g., 50/50 or skewed toward engagement) to tailor scoring to their business. Account scoring for ABM adds an optional intent component and allows configurable weighting among engagement, fit, and intent.[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)

## Multi‑channel and experimentation

The webinar highlights building multi‑channel journeys in Flow, such as: send an email, wait three days to check engagement, then for non‑engagers split traffic via a path experiment element into SMS vs WhatsApp paths to compare performance. Path experiments provide richer testing than a simple A/B test by allowing multiple branches and conditions; automatic winner selection is not yet available but is stated as a roadmap item.[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)

Growth’s pricing and usage model is consumption‑based; you pay in credits for segmentation, activation, and sending.[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)

The speakers explicitly say they have not heard any official plan to sunset Account Engagement or Marketing Cloud Engagement; both continue to receive features. Growth/Advanced is framed as an additional option rather than a replacement, and the choice among tools should be based on current use cases (e.g., need for business units, HTML‑heavy templates, ABM, reporting requirements) and timing relative to upcoming releases.[youtube](https://www.youtube.com/watch?v=OdhTwxPSjbU&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=2)

# 2
This page is a 1 hour 25 minute webinar that explains how Salesforce Data Cloud underpins Marketing Cloud on Core (“Marketing Cloud Growth/Advanced”), focusing on unified profiles, data modeling, segmentation, and activation for marketers.[youtube](https://www.youtube.com/watch?v=3hWARjc1c6c&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=3)

## Purpose and context

The session is #2 in a seven‑part Marketing Cloud Growth/Advanced series, aimed at marketers and architects who want to understand how to use Data Cloud as the data foundation for Marketing Cloud on Core, including setup concepts and practical tips. It includes a slide deck link and ends with an extended Q&A where Salesforce PMs and SEs answer real implementation questions.[youtube](https://www.youtube.com/watch?v=3hWARjc1c6c&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=3)

## Core data model and unification

The speaker contrasts legacy models (Account Engagement prospects, Marketing Cloud Engagement contacts/data extensions) with Marketing Cloud on Core’s **unified individuals** stored in Data Cloud. Data from Salesforce CRM (leads, contacts, prospects) and external sources (e‑commerce, ERP, files, S3 profiles) is ingested via Data Streams and Data Lake Objects, then harmonized into a standardized “Individual” DMO and further unified into “Unified Individuals.”[youtube](https://www.youtube.com/watch?v=3hWARjc1c6c&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=3)

Identity resolution uses match rules (e.g., first name + normalized email, or name + normalized phone) to decide when multiple individuals are the same person, and reconciliation rules (e.g., last‑updated wins, source priority) to decide which field values populate the unified record. The result is a single unified individual that aggregates profile attributes, contact points (email/phone/address), engagement, consent, and other related objects.[youtube](https://www.youtube.com/watch?v=3hWARjc1c6c&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=3)

## Data types, relationships, and calculated insights

Data Cloud stores not only profile data but also engagement events (email opens, clicks, form submissions, page views), consent, and arbitrary business data, all modeled as DMOs with relationships visualized in a graph view. Marketers can extend the model with custom DMOs/DSOs, add custom fields, and compute **calculated insights** (e.g., overall marketing score, lifetime value, number of rentals) using SQL expressions that become fields on the unified individual for segmentation and personalization.[youtube](https://www.youtube.com/watch?v=3hWARjc1c6c&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=3)

The session highlights **data graphs** as “bags of data” associated with unified individuals that expose selected related records (opportunities, campaigns, scores, contact points, etc.) to builders in Marketing Cloud on Core. These data graphs are then used by the email builder (for merge fields and variations) and Flow Builder (for decision splits and logic) to personalize content and paths.[youtube](https://www.youtube.com/watch?v=3hWARjc1c6c&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=3)

## Segmentation and activation

Within Data Cloud, marketers create segments of unified individuals using profile attributes, related objects (e.g., campaign member status), engagement, consent, or calculated insights. Segments can be built directly in Data Cloud, via quick filters in flows, or via AI‑assisted prompts, and are reusable in Marketing Cloud on Core, Marketing Cloud Engagement (as data extensions), Account Engagement (as dynamic lists), and ad platforms.[youtube](https://www.youtube.com/watch?v=3hWARjc1c6c&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=3)

The webinar stresses that to send emails in Marketing Cloud on Core today, segments must be defined on unified individuals, and that segments are not real‑time—they refresh on a schedule—similar to identity resolution, data graphs, and calculated insights, all of which have processing latency.[youtube](https://www.youtube.com/watch?v=3hWARjc1c6c&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=3)

## Builder usage and UI integration

In the email builder, marketers use the default data graph to drive conditional variations (dynamic content) and merge fields based on unified individual data and related objects. In Flow Builder, decision splits and flow logic read from a chosen data graph (overridable per‑flow) so that each step can branch based on scores, attributes, or related records for the current unified individual.[youtube](https://www.youtube.com/watch?v=3hWARjc1c6c&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=3)

On Salesforce CRM record pages (lead/contact/prospect), Data Cloud widgets show unified activity timelines, calculated insights, consent, and related records from Data Cloud, giving sales and service users a single view of engagement and marketing scores.[youtube](https://www.youtube.com/watch?v=3hWARjc1c6c&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=3)

## Credits and consumption model

The session briefly covers the Data Cloud credit model: different activities consume different credit types (ingestion, profile unification, calculated insights, segmentation/activation) with different multipliers per million rows, and batch vs streaming ingestion have different rates. Identity resolution is highlighted as particularly credit‑intensive, while calculated insights are comparatively lightweight, and users are encouraged to monitor usage via the “digital wallet”/credit card view in Data Cloud.[youtube](https://www.youtube.com/watch?v=3hWARjc1c6c&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=3)

## Implementation tips and demo

Practical guidance includes: design identity resolution early; filter data at the DSO level to limit which records participate in identity resolution; understand that formula fields are not retroactive; ensure connector permissions so custom Salesforce objects appear in Data Cloud; and design data graphs and calculated insights around planned segmentation and personalization use cases.[youtube](https://www.youtube.com/watch?v=3hWARjc1c6c&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=3)

A live demo walks through adding a custom “Boat Reservation” object from Salesforce into Data Cloud: creating a Data Stream, mapping it to a new DMO, relating it to Individuals via contact lookup, then segmenting unified individuals who had a reservation “yesterday” to send a post‑trip survey using a segment‑triggered flow and personalized email content. The session concludes with an extensive Q&A covering edge cases like multiple email addresses on a unified individual, account scoring enablement, deleting DMO fields, and roadmap items such as more granular targeting of specific contact points.[youtube](https://www.youtube.com/watch?v=3hWARjc1c6c&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=3)


# 3 

This video is a 72‑minute webinar on Marketing Cloud Growth/Advanced that deep‑dives into **Flow**, builders (forms, landing pages, email), personalization, and reporting in Marketing Cloud on Core, with live demos and Q&A at the end.[youtube](https://www.youtube.com/watch?v=YpRpKytV2EA&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=4)

## Flow in Marketing Cloud on Core

The speaker positions Flow as the “heartbeat” of Marketing Cloud on Core, replacing Engagement Studio for Account Engagement users and Journey Builder for Marketing Cloud Engagement users. Flows are always launched from the campaign record and serve as the engine for marketing execution and lead nurturing across the customer journey.[youtube](https://www.youtube.com/watch?v=YpRpKytV2EA&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=4)

Key capabilities demonstrated include sending emails/SMS/WhatsApp, adding wait steps (minutes to months), creating/updating Salesforce records on any object, branching based on engagement or attributes, debugging, and using path experiments (with Marketing Cloud Advanced). Two flow types are covered: segment‑triggered flows (scheduled runs over Data Cloud segments for campaigns like top‑of‑mind, re‑engagement, events) and event‑triggered flows (form submissions, email opens/clicks, WhatsApp subscriptions, etc.) that run continuously in the background once configured.[youtube](https://www.youtube.com/watch?v=YpRpKytV2EA&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=4)

The Flow Builder UI is explained: a central canvas for elements and connectors, a top bar for saving, debugging, and versioning, and a right‑hand properties panel to configure each element. The demo shows configuring start elements, segment schedules and republish behavior, and multiple element types: interaction (send message, consent request, subflows), logic (decision, path experiment, waits by time/date/event), and data (create/get/update/delete records, including deduping by email before create).[youtube](https://www.youtube.com/watch?v=YpRpKytV2EA&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=4)

## Forms and landing pages

The session then moves to builders for forms and landing pages in Marketing Cloud on Core. Forms are used to capture user info and create or update Salesforce records (e.g., Lead/Contact), and can be embedded on authenticated domains via iframes while being automatically linked to landing pages, campaigns, and flows for automation.[youtube](https://www.youtube.com/watch?v=YpRpKytV2EA&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=4)

Form creation is shown two ways: via a signup form event‑triggered campaign template (which auto‑creates form + flow) or via the Content tab and CMS “Add content → Form.” Best practices demonstrated include adding at least one input and one button, mapping to lead/contact via data sources (so object fields are available to drag onto the canvas), configuring submission actions (thank you message vs redirect URL), styling with brand plus layout controls, and publishing while ensuring connection to a flow and campaign.[youtube](https://www.youtube.com/watch?v=YpRpKytV2EA&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=4)

Landing pages are presented as places to host content, collect leads, and promote campaigns, optionally including forms or serving as “thank you” pages. The builder supports layouts, images, headings, lists, SEO metadata (meta description, index settings, additional head code), desktop/mobile preview, and vanity URLs. The speaker stresses that vanity URL aliases must be set before first publish and cannot be changed afterward.[youtube](https://www.youtube.com/watch?v=YpRpKytV2EA&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=4)

## End‑to‑end signup and invite flows (demos)

A substantial live demo builds an end‑to‑end signup process for a Marketing Cloud on Core bootcamp and a nurture invite flow. For the signup flow, an event‑triggered flow fires on form submission, then uses “Create or Update Lead” with field mapping and a match rule on email so existing leads are updated rather than duplicated.[youtube](https://www.youtube.com/watch?v=YpRpKytV2EA&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=4)

The flow then uses a Get Records element to fetch the specific Salesforce Campaign by name/ID and stores fields to use downstream. A Create Records element creates campaign member records, assigning the campaign ID from Get Records and lead ID from the prior create/update step, and hard‑codes member status to “Registered.” Finally, a consent action sets consent status to opt‑in for the contact point email, channel email, and a “marketing” communication subscription so registrants can receive marketing communications.[youtube](https://www.youtube.com/watch?v=YpRpKytV2EA&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=4)

A second demo shows a segment‑triggered invite flow for bootcamp invitations with prebuilt schedule, segment selection, initial email, and wait steps. They extend it: after the first invite email, they add a “Wait until event” that listens for an email link click to the signup landing page within a defined time. A Decision element then checks via the data graph and campaign member data whether the recipient has a campaign member status of “Registered”; registrants exit, while non‑registrants go down the null path and receive a reminder invitation email.[youtube](https://www.youtube.com/watch?v=YpRpKytV2EA&list=PL_o00w2Z5LYGQoG4TWuRAfw9SaDhLVqk0&index=4)

## Personalization and dynamic content
